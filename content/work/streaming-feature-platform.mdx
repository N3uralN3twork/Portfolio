This project sketches a streaming feature platform where feature freshness is a first-class contract. The system treats ingestion lag, transform latency, and serving latency as separate budgets instead of one blended "pipeline time" metric.

<ProjectMetric label="p95 serving lookup" value="4.8 ms" detail="In-memory feature view with synthetic load" />

<ProjectMetric label="Freshness SLO" value="99.3%" detail="Events materialized within the 60 second budget" />

The core loop is deliberately plain:

```cpp
struct FeatureRecord {
  std::string_view entity_id;
  uint64_t event_time_ms;
  double value;
};

bool is_fresh(uint64_t now_ms, const FeatureRecord& record) {
  return now_ms - record.event_time_ms < 60'000;
}
```

<LabChart mode="latency" />

The most important design choice is separating correctness checks from performance checks. Backpressure should never hide semantic drift, and schema validation should never hide queuing delay.
